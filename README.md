# Centurio

**Персональная, всегда доступная из трея «горячая панель» приложений.**

Centurio — это не просто лаунчер, а личный «пульт управления» рабочим столом. Вместо
хаоса из ярлыков и папок на рабочем столе вы один раз раскладываете программы по
смысловым категориям (Работа, Игры, Разработка, Творчество…), а дальше запускаете их
в один клик из чистой, всегда знакомой по расположению панели, которая живёт в системном
трее и открывается мгновенно.

Аналогия — по ощущению, а не по функциям — лёгкая оверлей-панель NVIDIA App: не мешает,
не занимает место в панели задач, но всегда под рукой.

---

## Возможности

Реализованы все функции из технического задания и дизайна:

- **Быстрый запуск приложений** — добавляйте любую программу (выбор `.exe`/файла через
  системный диалог) и запускайте её в один клик или горячей клавишей.
- **Категории и группировки** — организуйте приложения по смыслу; создавайте,
  переименовывайте и удаляйте категории; при удалении приложения переносятся в первую.
- **Сворачивание в трей** — значок в трее, клик открывает/прячет окно, меню «Открыть /
  Выход». Настраивается поведение кнопок «свернуть» и «закрыть».
- **Автозапуск с Windows** — переключатель запуска вместе с системой (скрытно, в трей).
- **Избранное** — отмечайте часто используемые приложения звёздочкой.
- **Недавние и «Продолжить»** — история запусков и hero-карточка последнего приложения.
- **Быстрый запуск (Quick launch)** — закреплённый ряд с глобальными горячими клавишами
  (`Ctrl+1…9`).
- **Индикатор «Запущено»** — Centurio отслеживает запущенные им процессы.
- **Поиск** (`Ctrl+K`), сортировка, режимы «сетка/список», настройка акцентного цвета и
  размера плиток.

## Как это выглядит

Тёмный интерфейс из дизайна: кастомная строка заголовка, левый рельс категорий, боковая
панель фильтров (Все / Избранное / Недавние / Запущено), основная область с hero-карточкой,
рядом быстрого запуска и плитками приложений по категориям, строка статуса.

## Технологии

- **Electron** — фреймворк для десктопного приложения (трей, автозапуск, запуск программ,
  глобальные горячие клавиши, безрамочное окно).
- Рендерер — vanilla JS + CSS без сборки, точно повторяющий дизайн.
- Хранилище — локальный JSON-файл в каталоге пользователя (`userData`).

## Запуск

```bash
npm install      # установит Electron
npm start        # запустит приложение
```

Иконки приложения/трея генерируются скриптом без внешних зависимостей:

```bash
npm run gen-icon
```

Проверка синтаксиса всех модулей:

```bash
npm run check
```

### Предпросмотр интерфейса в браузере

Файл `src/renderer/index.html` можно открыть в обычном браузере — при отсутствии Electron
подключается встроенный мок (`demo-data.js`) с демо-библиотекой из дизайна, и интерфейс
полностью интерактивен (для дизайн-ревью и скриншотов).

## Сборка дистрибутива

```bash
npm run dist     # electron-builder: Windows (nsis), macOS (dmg), Linux (AppImage)
```

## Структура проекта

```
src/
  main/                 # процесс Electron (main)
    main.js             # окно, IPC, трей, автозапуск, глобальные хоткеи
    preload.js          # безопасный мост window.centurio (contextBridge)
    store.js            # персистентность (apps / categories / settings)
    launcher.js         # запуск программ и отслеживание «Запущено»
    tray.js             # значок в трее и меню
  renderer/             # интерфейс (renderer)
    index.html
    styles.css          # тёмная тема из дизайна
    renderer.js         # построение UI и вся логика взаимодействия
    demo-data.js        # мок-мост + демо-данные для браузерного предпросмотра
assets/                 # иконки (генерируются scripts/gen-icon.js)
scripts/gen-icon.js     # генератор PNG-иконок без зависимостей
```

## Архитектура

Процесс `main` владеет состоянием (через `store.js`) и системными возможностями
(трей, автозапуск, запуск процессов, хоткеи). Рендерер общается с ним только через
явный API `window.centurio`, объявленный в `preload.js` (`contextIsolation: true`,
`nodeIntegration: false`). Любое изменение состояния в main транслируется в рендерер
событием `state:update`, и интерфейс перерисовывается из единого источника правды.
