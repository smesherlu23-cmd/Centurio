# Centurio

**Персональная, всегда доступная из трея «горячая панель» приложений.**

Centurio — это не просто лаунчер, а личный «пульт управления» рабочим столом. Вместо
хаоса из ярлыков и папок на рабочем столе вы один раз раскладываете программы по
смысловым категориям (Работа, Игры, Разработка, Творчество…), а дальше запускаете их
в один клик из чистой, всегда знакомой по расположению панели, которая живёт в системном
трее и открывается мгновенно.

Аналогия — по ощущению, а не по функциям — лёгкая оверлей-панель NVIDIA App: не мешает,
не занимает место в панели задач, но всегда под рукой.

Приложение написано на **Flet (Python)**.

---

## Возможности

Реализованы все функции из технического задания и дизайна:

- **Быстрый запуск приложений** — запускайте программы в один клик или горячей клавишей.
- **Добавление без ручного поиска** — Centurio сам находит установленные приложения
  (ярлыки меню «Пуск», «Программы и компоненты» и App Paths в Windows; `.desktop` в
  Linux; `.app` в macOS) и показывает готовый список: выбираете нужное — оно добавляется
  автоматически, без ввода названия и описания. Системные утилиты отфильтрованы. Для
  программ вне списка остаётся выбор файла вручную (имя подставляется из файла).
- **Игры** — находятся установленные игры Steam (по `appmanifest`, запуск через
  `steam://…`) и Epic Games (по манифестам лаунчера).
- **Настоящие иконки приложений** — Centurio показывает иконки, извлечённые из самих
  программ (иконка `.exe` в Windows, тема значков в Linux, `.icns` в macOS, обложки Steam);
  если иконку найти не удалось — цветная плитка с буквой.
- **Категории и группировки** — организуйте приложения по смыслу; создавайте,
  переименовывайте и удаляйте категории; при удалении приложения переносятся в первую.
- **Сворачивание в трей** — значок в трее, меню «Открыть / Выход». Настраивается поведение
  кнопок «свернуть» и «закрыть».
- **Автозапуск с Windows** — переключатель (реестр `HKCU…\Run`; на Linux/macOS — файл
  автозапуска / LaunchAgent).
- **Избранное**, **Недавние**, hero-карточка «Продолжить».
- **Быстрый запуск (Quick launch)** — закреплённый ряд с горячими клавишами `Ctrl+1…9`.
- **Индикатор «Запущено»** — Centurio отслеживает запущенные им процессы.
- **Поиск** (`Ctrl+K`), сортировка, режимы «сетка/список», акцентный цвет, размер плиток.

## Как это выглядит

Тёмный интерфейс из дизайна: кастомная строка заголовка, левый рельс категорий, боковая
панель фильтров (Все / Избранное / Недавние / Запущено) с переключателями автозапуска и
трея, основная область с hero-карточкой, рядом быстрого запуска и плитками приложений по
категориям, строка статуса. Типографика — Inter (шрифты встроены, приложение
самодостаточно и не зависит от системных шрифтов).

## Запуск

```bash
python -m venv .venv
source .venv/bin/activate      # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

Значок приложения и трея генерируется при первом запуске скриптом без внешних зависимостей
(`app/iconify.py`). Отдельно:

```bash
python -m app.iconify
```

### Предпросмотр интерфейса в браузере

Для дизайн-ревью интерфейс можно запустить в браузере (стартует с пустой
библиотекой — добавьте приложения кнопкой «Добавить приложение»):

```bash
CENTURIO_WEB=1 python main.py
# откройте http://localhost:8550
```

## Сборка дистрибутива

Flet собирает нативные приложения (нужен установленный Flutter SDK):

```bash
flet build windows      # или macos / linux
# либо однофайловая сборка:
flet pack main.py --name Centurio --icon assets/icon.png --add-data "assets:assets"
```

## Структура проекта

```
main.py                # точка входа: окно, трей, автозапуск, горячие клавиши
app/
  store.py             # персистентность (apps / categories / settings) в JSON
  launcher.py          # запуск программ и отслеживание «Запущено»
  discovery.py         # авто-поиск установленных приложений (Пуск/.desktop/.app)
  autostart.py         # автозапуск (Windows реестр / Linux .desktop / macOS plist)
  tray.py              # значок в трее (pystray, мягко отключается при отсутствии)
  colors.py            # палитра и per-app градиенты плиток
  iconify.py           # генератор PNG-иконок без зависимостей
  ui.py                # построение интерфейса Flet и вся логика взаимодействия
  dialogs.py           # окна: добавить/изменить приложение, категории, настройки
assets/fonts/          # встроенные шрифты (Inter + моно)
requirements.txt
```

## Архитектура

Один объект `CenturioUI` владеет страницей Flet, строит «оболочку» один раз и
перерисовывает динамические области (рельс, боковую панель, контент, строку статуса)
при каждом изменении состояния. Состояние (приложения, категории, настройки) живёт в
`Store` и пишется в JSON атомарно. Системные возможности — запуск процессов, трей,
автозапуск — вынесены в отдельные модули и подключаются через контроллеры, поэтому UI
не зависит от платформы и одинаково работает на десктопе и в веб-предпросмотре.

Данные хранятся в пользовательском каталоге: `%APPDATA%\Centurio\centurio-data.json`
(Windows), `~/.local/share/Centurio/…` (Linux), `~/Library/Application Support/Centurio/…`
(macOS).
